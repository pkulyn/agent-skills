# 文体模板3：汇报材料

> 适用范围：向上级汇报工作、专项工作汇报、调研汇报
> 模型基础：三层精简结构变体
> 参考样本：《政府工作汇报范文10篇》

---

## 一、整体结构

```
开头（5%）
├── 汇报背景+总体情况

第一部分：主要成效/基本情况（35-40%）
├── 一、主要成效
│   ├── （一）成效1：做法+结果
│   ├── （二）成效2：做法+结果
│   └── （三）成效3：做法+结果

第二部分：主要做法/工作经验（30-35%）
├── 二、主要做法
│   ├── （一）做法1：[具体措施]
│   ├── （二）做法2：[具体措施]
│   └── （三）做法3：[具体措施]

第三部分：存在问题及建议（20-25%）
├── 三、存在问题和建议
│   ├── （一）问题1+建议1
│   ├── （二）问题2+建议2
│   └── （三）问题3+建议3

结尾（3-5%）
├── 表态+请求
```

**与工作总结的关键区别**：
- 汇报材料更突出**做法**（怎么做的），而不仅是成绩
- 增加了**建议**部分（向上级请求支持或建议政策调整）
- 语言更简洁，面向上级、便于口头汇报
- 可配有PPT或图表

## 二、各部分写作要点

### 2.1 开头

**标准格式**：
```
尊敬的[领导职务]：
现将[单位][工作名称]情况汇报如下：
```

**简要版**（5分钟以内汇报）：
```
[单位]自[时间]以来，围绕[主题]，
[总体做法概括]，[总体成效概括]。
```

**详细版**（10分钟以上汇报）：
```
[领导职务]：
按照[上级文件/会议]要求，[单位]高度重视，
迅速行动，[总体情况1-2句]。现将有关情况汇报如下：
```

**写作要点**：
- 极简，1-2段
- 交代汇报缘由
- 点明总体判断

### 2.2 第一部分：主要成效

**两种组织方式**：

#### 方式1：按成效类型分（推荐）
```
一、主要成效

（一）[指标/领域1]明显提升。
    [具体数据]→[对比]→[判断]

（二）[指标/领域2]取得突破。
    [具体数据]→[对比]→[判断]

（三）[指标/领域3]稳步推进。
    [具体数据]→[对比]→[判断]
```

#### 方式2：按工作板块分
```
一、主要成效

（一）[板块1]方面：[做法]→[结果]
（二）[板块2]方面：[做法]→[结果]
（三）[板块3]方面：[做法]→[结果]
```

**数据展示技巧**：
```
❌ 指标完成情况较好
✅ [指标]达到[X]，同比增长[X]%，超额完成年度目标[X]个百分点

❌ 覆盖面不断扩大
✅ 从[X]个扩大到[X]个，覆盖率由[X]%提升至[X]%

❌ 质量明显改善
✅ [质量指标]从[X]提升至[X]，排名从第[X]位升至第[X]位
```

**写作要点**：
- 每个成效2-3段：定性→定量→判断
- 成效要有对比（同比/环比/与目标比）
- 突出亮点，一般成效合并表述

### 2.3 第二部分：主要做法

**核心原则**：让上级知道"怎么做到的"，具有可推广性

**标准展开模板**：
```
（一）[做法标题]：[1句概括]。

[具体措施1]：[内容]
[具体措施2]：[内容]
[具体措施3]：[内容]

[效果简述1句]。
```

**做法标题命名技巧**：
```
✅ "建立[X]机制，确保[X]"        → 机制+效果
✅ "突出[X]导向，推动[X]"        → 方法+目标
✅ "坚持[X]标准，实现[X]"        → 标准+结果
✅ "创新[X]模式，提升[X]"        → 创新+提升

❌ "关于[X]的做法"               → 太泛
❌ "认真做好[X]工作"             → 太平
❌ "领导高度重视"                → 太空
```

**常见做法类型**：
1. **组织领导型**：成立专班、明确分工、责任到人
2. **制度建设型**：建章立制、规范流程、完善机制
3. **创新方法型**：创新模式、引入技术、优化流程
4. **协同推进型**：部门联动、上下联动、多方参与
5. **督查考核型**：定期调度、考核评估、追责问责

### 2.4 第三部分：存在问题及建议

**两种组织方式**：

#### 方式1：问题和建议分开
```
三、存在问题和建议

（一）存在问题
1. [问题1]：[具体表现]
2. [问题2]：[具体表现]
3. [问题3]：[具体表现]

（二）几点建议
1. [建议1]：[具体内容]
2. [建议2]：[具体内容]
3. [建议3]：[具体内容]
```

#### 方式2：问题和建议一一对应（推荐）
```
三、存在问题及建议

（一）关于[方面1]的问题和建议
    问题：[具体表现]
    建议：[对策建议]

（二）关于[方面2]的问题和建议
    问题：[具体表现]
    建议：[对策建议]

（三）关于[方面3]的问题和建议
    问题：[具体表现]
    建议：[对策建议]
```

**建议的类型**：
1. **请求支持型**："恳请上级[给予支持/协调解决]"
2. **政策建议型**："建议[调整/完善/出台]相关政策"
3. **经验推广型**："建议[推广/借鉴]XX做法"

**写作要点**：
- 问题2-4个，建议与问题对应
- 建议要具体、可操作
- 不提分外之事，不推分内之责

### 2.5 结尾

**标准格式**：
```
以上汇报，不当之处，请[领导职务]批评指正。
```

**表态版**：
```
下一步，我们将认真落实[领导]指示要求，
[表态1句]，[承诺1句]，
确保[目标]。
```

## 三、汇报材料特殊场景

### 3.1 口头汇报版 vs 书面汇报版

| 维度 | 口头版 | 书面版 |
|------|--------|--------|
| 篇幅 | 1500-3000字 | 3000-8000字 |
| 数据 | 3-5个核心数据 | 充分展示 |
| 结构 | 更简明 | 更完整 |
| 建议 | 1-2条重点 | 3-5条系统 |
| 语言 | 口语化 | 规范化 |

### 3.2 限时汇报的结构调整

| 时间 | 结构 | 要点 |
|------|------|------|
| 3分钟 | 成效1段+做法1段+建议1段 | 只讲最亮点的 |
| 5分钟 | 成效2-3点+做法2-3点+建议1-2点 | 每点1句话 |
| 10分钟 | 完整三段式 | 每点2-3句话 |
| 15分钟+ | 完整三段式+案例 | 可展开具体案例 |

## 四、常见问题与避坑

| 问题 | 表现 | 解决方案 |
|------|------|----------|
| 只报喜不报忧 | 通篇成绩无问题 | 必须有2-3个问题 |
| 建议提不上去 | "恳请上级重视" | 写具体请求事项 |
| 做法太笼统 | "加强领导、精心组织" | 写具体措施和路径 |
| 成效缺数据 | "取得显著成效" | 每个成效配数据 |
| 超时 | 领导不耐烦 | 严格控制篇幅和要点数 |

## 五、自检清单

- [ ] 开头是否交代汇报缘由和总体判断
- [ ] 成效是否有数据对比（至少3处）
- [ ] 做法是否有具体措施（非空话）
- [ ] 问题和建议是否一一对应
- [ ] 建议是否具体可操作
- [ ] 篇幅是否适配汇报时间
- [ ] 关键数据是否核实准确
- [ ] 语言是否适合口头表达（如需口头汇报）
