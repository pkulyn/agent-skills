# 质量检查清单

## 客观检查项（替代主观评分）

| 检查项 | 标准 | 不合格条件 |
|--------|------|-----------|
| frontmatter完整性 | 含title/date/source_hash/tags | 缺少任一字段 |
| frontmatter格式 | type使用中文规范值、日期无引号、tags数组格式 | type用英文值或日期加引号 |
| 关联实体数量 | 摘要3-8个 | <3 或 >8 |
| 关联主题数量 | 摘要3-5个 | <3 或 >5 |
| 双链有效性 | 所有[[链接]]指向存在的页面 | 有断链 |
| 命名规范 | 文件名符合YYYY-MM-DD_{名称}.md | 不符合 |
| 章节完整性 | 包含模板要求的所有章节 | 缺少核心章节 |
| 实体去重 | 无同名或别名重复实体 | 存在重复 |
| 结构一致性 | 章节无重复，顺序正确，标题统一中文 | 有重复章节或英文标题 |

**评级**：
- A（优秀）：9/9项通过
- B（良好）：7-8项通过
- C（合格）：5-6项通过
- D（不合格）：<5项通过

## index.md统计更新规则

更新index.md时，统计数据必须通过Glob扫描实际计算，**禁止使用"30+""21+"等模糊表示**：

1. 对`Wiki/entities/`执行Glob扫描，获取实体精确数量
2. 对`Wiki/summaries/`执行Glob扫描，获取摘要精确数量
3. 对`Wiki/topics/`执行Glob扫描，获取主题精确数量
4. 将精确数字写入统计表

## 执行检查清单

### 每次任务完成后

- [ ] 使用了正确的YAML frontmatter（含source_hash字段）
- [ ] frontmatter格式一致：type字段使用中文规范值、日期不加引号、tags使用YAML数组格式
- [ ] 应用了适当的模板，所有章节标题使用中文
- [ ] 添加了到相关页面的交叉引用
- [ ] 更新了`Wiki/log.md`（含source_hash和source_mtime）
- [ ] 如果结构发生变化，更新了`Wiki/index.md`（统计数据通过Glob精确计算）
- [ ] 使用了从知识库根目录开始的相对路径
- [ ] 验证了所有Wiki链接的有效性
- [ ] 执行了文件路径双重校验
- [ ] 更新了`Wiki/.state/registry.json`
- [ ] 实体别名检测已执行

### 批量任务额外检查项

- [ ] 已创建快照目录和manifest.json
- [ ] 已执行快照创建校验
- [ ] 已验证幂等性
- [ ] 已应用正确的冲突处理策略
- [ ] merge操作的实体页面已执行整理规则
- [ ] 已生成详细的批量处理报告（含质量检查清单结果）
- [ ] 已向用户提供回滚选项
- [ ] 已更新index.md中的统计数据（精确数字）
- [ ] 失败文件已记录到`Wiki/.state/errors/`

## 交叉引用规则

- **内部Wiki链接**：`[[页面名称]]` 或 `[[页面名称|显示文本]]`
- **来源引用**：`[[../folder/source.md]]`
- **章节链接**：`[[页面名称#章节]]`
- **块链接**：`[[页面名称#^block-id]]`
- **批量任务引用**：`[[log.md#{任务ID}]]` 或 `[[Wiki/reports/batch-{任务ID}.md]]`
