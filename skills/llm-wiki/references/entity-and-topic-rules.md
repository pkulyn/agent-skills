# 实体与主题提取规则

## 实体类型规范

| 类型值 | 英文对照 | 适用范围 | 示例 |
|--------|----------|----------|------|
| 人物 | person | UP主、开发者、研究者 | 悟鸣、Andrej Karpathy |
| 组织 | organization | 公司、团队、社区 | 润泽科技、Anthropic |
| 工具 | tool | 软件、平台、插件、Skill | OpenClaw、Claude Code、MCP协议 |
| 概念 | concept | 理论、方法、框架 | 12-Factor AgentOps、社会权力 |
| 地点 | location | 地理位置相关 | - |
| 事件 | event | 会议、发布、里程碑 | - |
| 武器装备 | weapon | 军事装备 | F-22战斗机 |

**强制规则**：
- frontmatter中`type`字段必须使用上述中文类型值之一，禁止使用英文值
- 日期字段（created/updated）不加引号，格式为`YYYY-MM-DD`
- tags字段使用YAML数组格式`[entity, 类型标签]`

## 实体提取规则

1. **重要性原则**：仅提取与内容核心相关的重要实体，忽略次要、边缘的实体
2. **类型明确**：每个实体必须分类为7种类型之一，使用中文类型值
3. **一致性原则**：使用已存在的实体名称，避免创建意思相同但名称不同的重复实体
4. **避免主题混淆**：实体是具体对象，主题是分类范畴
5. **数量限制**：每个摘要提取3-8个实体
6. **去重合并原则**（批量处理时必执行）：创建前用`Glob`扫描查重，同名直接合并，别名按检测规则合并

## 实体别名检测规则（创建前必执行）

1. **前缀/后缀变体检测**：去除常见后缀（AI、App、工具、平台、协议等）后比较；去除常见前缀（教授、老师、博士等）
2. **中英文名对应检测**：已知映射表（Claude Code ↔ openclaw等）
3. **Grep内容扫描**：扫描`Wiki/entities/`检查是否被已有实体提及
4. **合并执行**：不创建新实体，在已有实体中追加新来源，添加`aliases`字段

## 实体页面整理规则（merge时必执行）

1. **去重整理**：合并重复条目
2. **章节排序**：定义→详细信息→补充视角→相关实体→相关主题→来源
3. **补充视角压缩**：超过5个时智能压缩，仍超5个时将最早的归档到折叠区域
4. **结构校验**：检查符合实体模板结构
5. **frontmatter更新**：更新`updated`字段，添加`aliases`字段（如有）

## 主题提取规则

1. **必须基于内容提取**：禁止使用默认值或复用不相关的主题
2. **层级原则**：优先提取最具体的主题，再提取更宽泛的上层主题
3. **数量限制**：每个摘要关联3-5个主题
4. **一致性原则**：使用已存在的主题名称，避免创建意思相同但名称不同的重复主题
5. **避免实体混淆**：主题是分类范畴，不是具体实体
6. **语义匹配度校验**（INGEST步骤7中即时执行）：核心关键词匹配、反向验证、禁止领域跳跃
7. **主题层级校验**：至少关联1个具体子主题和1个宽泛父主题

**常见主题分类参考**：
- 哲学类：哲学研究、马克思主义哲学、西方哲学、中国哲学
- 技术类：AI辅助开发、Agent工程化、知识管理、远程开发
- 社会科学：社会学、经济学、政治学、历史学
- 自然科学：物理学、生物学、化学、数学
- 人文艺术：文学、艺术、音乐、电影、历史
